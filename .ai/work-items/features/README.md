# `.ai/work-items/features/` — Features

Feature work items: new capability within the existing (or planned) architecture. Scoped phases, dependency-ordered.

## Typical shape

Scope → design within architecture → implement → test (per `../../system/TESTING_SELECTION_RULES.md`) → review → integrate. Feature phases are generated dynamically (`../../system/PHASE_GENERATION_RULES.md`), not from a fixed template.

## Each feature records

- Title & goal, link to project/phase.
- Applications/areas affected (`../../system/APPLICATION_SELECTION_RULES.md`).
- Tasks with acceptance criteria (`../../system/TASK_GENERATION_RULES.md`).
- Required skills; testing plan; risks/behavior impact.
