# `.ai/memory/` — Durable Reusable Decisions

Concise, durable memory: decisions, conventions, and resolved conflicts that should persist across tasks and sessions. Keep it **small and high-signal** (`../system/TOKEN_OPTIMIZATION_RULES.md`).

## What belongs here

- Cross-task decisions and their rationale (the "why").
- Resolved rule conflicts likely to recur (`../system/README.md` precedence).
- Standing user preferences about how work should be done.
- Conventions adopted for this repo/project.

## What does not belong here

- Per-task status → `../work-items/`, `../projects/current/`.
- Large domain/architecture docs → `../knowledge/`.
- Generated reports → `../generated/`.
- Anything already canonical in `../system/`.
