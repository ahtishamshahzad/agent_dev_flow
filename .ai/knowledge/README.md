# `.ai/knowledge/` — Durable Domain & Architecture Knowledge

Durable knowledge for the **current work**: domain concepts, architecture decisions, glossary, constraints, and integration details that agents need to reason correctly. Distinct from `../references/` (external-facing material) and `../memory/` (concise reusable decisions).

## What goes here

- Domain glossary and business rules relevant to the project.
- Architecture notes and decision records (ADR-style) for the current system.
- Integration details (third-party services, contracts) worth persisting.
- Non-obvious constraints that shape implementation.

## What does not go here

- Canonical system rules → `../system/`.
- Per-task status → `../work-items/` and `../projects/current/`.
- Generated reports → `../generated/`.
- External reference docs → `../references/`.
